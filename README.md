# OS.nix

Single flake for:

- NixOS (bare metal + Parallels VMs)
- macOS via nix-darwin
- Shared Home Manager setup

## Features

### NixOS

- Sway / i3
- Basic development tools
- Home Manager
- Parallels Tools from `nixpkgs-unstable` (when building Parallels hosts)

### macOS (nix-darwin)

- nix-darwin (following `nixpkgs` 25.05)
- Home Manager
- Host config in `./hosts/darwin/mbp.nix`

## Hosts / Flake Outputs

The flake defines these configurations:

### NixOS (`nixosConfigurations`)

| Flake name      | System        | Host key        | Notes                    |
|-----------------|---------------|-----------------|--------------------------|
| `nixos-x86_64`  | x86_64-linux  | `nixos-x86_64`  | Bare metal / generic     |
| `prl-x86_64`    | x86_64-linux  | `prl-x86_64`    | Parallels VM             |
| `nixos-aarch64` | aarch64-linux | `nixos-aarch64` | Bare metal / generic ARM |
| `prl-aarch64`   | aarch64-linux | `prl-aarch64`   | Parallels VM (Apple SoC) |

Parallels-specific bits (Parallels Tools, etc.) are enabled when `isParallels = true` via `./modules/prl-tools.nix`.

### macOS (`darwinConfigurations`)

| Flake name   | System         | Host key     | Notes        |
|--------------|----------------|--------------|--------------|
| `darwin-mbp` | aarch64-darwin | `darwin-mbp` | macOS laptop |

## Username and hostnames

The flake includes a placeholder `./local/default.nix` so the flake evaluates,  
but **you must replace its contents with your own values**.

Edit `./local/default.nix`:

```nix
{
  # Your local username
  userName = "your-user";

  # Hostnames for each flake configuration
  hosts = {
    nixos-x86_64 = "your-nixos-host";
    prl-x86_64 = "your-parallels-host";
    nixos-aarch64 = "your-arm-host";
    prl-aarch64 = "your-arm-parallels-host";
    darwin-mbp = "your-mac-hostname";
  };
}
```

> **Note:** The placeholder file only exists so the flake can evaluate.  
> You *must* replace the values before building any system.

## NixOS: hardware configuration

> **IMPORTANT:** Replace the hardware config on each NixOS install.

This repo includes a minimal placeholder at:

```text
hosts/nixos/hardware-configuration.nix
```

After installing NixOS on a machine, generate the real hardware config:

```sh
sudo nixos-generate-config --show-hardware-config \
  > hosts/nixos/hardware-configuration.nix
```

If you’re deploying a Parallels VM, remove any autogenerated Parallels-related entries such as:

```nix
hardware.parallels.enable = true;
nixpkgs.config.allowUnfreePredicate = ...;
```

Parallels tools are handled by the repo’s own module:

```text
./modules/prl-tools.nix
```

## Caveats (NixOS)

- Needs `WLR_NO_HARDWARE_CURSORS=1` for proper cursor rendering under Sway in Parallels
- Parallels Tools is unfree
- Only tested on:
  - Parallels VM (aarch64 / Apple Silicon)
  - Other setups are **not** guaranteed to work yet (x86_64, non-Parallels, etc.)

## Commands

### NixOS rebuild

```sh
# Parallels aarch64 (Apple Silicon VM)
sudo nixos-rebuild switch --flake .#prl-aarch64

# Bare metal aarch64
sudo nixos-rebuild switch --flake .#nixos-aarch64

# x86_64 hosts (examples)
sudo nixos-rebuild switch --flake .#nixos-x86_64
sudo nixos-rebuild switch --flake .#prl-x86_64
```

### macOS (nix-darwin) rebuild

From macOS:

```sh
sudo darwin-rebuild switch --flake .#darwin-mbp
```

### Garbage collection / optimisation

```sh
# Delete old generations (system)
sudo nix-env --delete-generations old --profile /nix/var/nix/profiles/system

# Garbage collect
sudo nix-collect-garbage -d

# Optimize store
sudo nix-store --optimise
```

## Notes

- `home-manager` is wired in via the flake and passed through `specialArgs`; host modules under `./hosts/nixos/` and `./hosts/darwin/` are expected to import and configure Home Manager as needed.
- Parallels-specific config is only injected for `prl-*` hosts via `isParallels = true`.
